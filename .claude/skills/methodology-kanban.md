# Kanban Methodology

Continuous flow. No sprints. Pull work when you have capacity. Finish before starting.

---

## Board Columns

Track work in PROGRESS.md using these sections:

- **## Backlog** — All known work, roughly prioritized (most important first)
- **## In Progress** — Currently being worked on (WIP limit: 3 items max)
- **## Done** — Completed work (with completion date)

## Core Rules

1. **WIP limit is 3** — Never have more than 3 items in progress. Finish something before starting something new. This is the single most important rule.
2. **Pull, don't push** — When you finish an item, pull the next highest-priority item from the backlog.
3. **Top of backlog = next up** — Keep the backlog ordered so the top item is always the most important thing to do next.

## Workflow

1. Pick the top item from ## Backlog
2. Move it to ## In Progress
3. Work on it until done
4. Run through the appropriate quality gate (Gate 2 minimum)
5. Move it to ## Done with today's date
6. Pull the next item

## Keeping the Board Healthy

- **Weekly**: Review the backlog. Remove stale items. Add new ones. Re-prioritize.
- **When stuck**: If an item is blocked for more than a day, note the blocker and consider pulling a different item (but respect the WIP limit).
- **Large items**: If something has been in progress for more than 3 days, it's too big. Break it into smaller pieces.

## When to Use Kanban

- Maintenance and bug-fix work
- Ongoing projects without clear milestones
- Support and operations tasks
- When sprint ceremonies feel like overhead
